## 정보디자인 기말 프로젝트 – 기획 내용 정리

### 1. 프로젝트 한줄 정의
- **목표**: 사용자가 직접 설정한 **가중치**를 기반으로 여러 블록체인의 **안정성을 비교·분석**하고, “지금 나에게 맞는 블록체인은 무엇인가?”를 시각적으로 보여주는 웹 서비스.

### 2. 핵심 질문 & 타깃
- **핵심 질문**
  - 블록체인의 안정성에 영향을 주는 핵심 요소는 무엇인가?
  - 이 요소들을 어떻게 점수화·가중치화해서 체인 간 비교를 할 것인가?
- **타깃 사용자**
  - 블록체인 입문자 또는 분석이 필요한 투자자
  - “어떤 체인이 더 안전한지, 어떻게 분석해야 하는지” 헷갈리는 사용자

### 3. 안정성 지표 구조 (3개의 축)

- **활동성(Activity)**
  - 제안 투표 참여도
  - 제안 수
  - IBC Out

- **지속성(Sustainability / Persistence)**
  - 코인 생존 시간
  - Nakamoto coefficient (NC)
  - 스테이킹 비율

- **경제적 변동성(Economic Volatility)**
  - 가격 변동성
  - 거래자 수
  - 보유 지분(시가총액 등)
  - 시간에 따른 트랜잭션 수

> 위 세 축 각각은 0–100 점 스케일로 정규화하고, 사용자가 **가중치 슬라이더/숫자 입력**을 통해 “나에게 중요한 축”을 조절하는 구조를 목표로 한다.

### 4. 사용할 데이터 속성(예시)
- **Base / Summary 정보**
  - `marketChange24h`, `marketChangePercentage24h` (단기 가격 변동성)
  - `marketCap` (시가총액, 보유 지분의 절대 규모)
  - `activeAccountCount` (활성 계정 수)
- **시계열 데이터**
  - `marketpricechangeper` (기간별 가격 변화율)
  - `txCount` (Block Transaction Count, 시간에 따른 트랜잭션 수)
- **거버넌스·검증자 관련**
  - proposal의 `voteAbstain` 등 (투표 참여/불참 데이터)
  - validator의 `votingPower`, `timestamp` (Nakamoto coefficient 계산용)

### 5. 예상 결과물(화면 구성)

- **① 가중치 컨트롤러**
  - 활동성 / 경제적 변동성 / 지속성에 대한 가중치 슬라이더 또는 숫자 입력
  - 사용자가 입력한 가중치를 시각화하는 그래프(도넛/막대 등)

- **② 블록체인 안정성 추이 그래프**
  - x축: 시간, y축: 안정성 종합 점수(1–100) 및 세부 점수
  - 기능:
    - 블록체인 선택(드롭다운, 랭킹 보드와 연동)
    - 기간 선택(1일 / 1주 / 1달 등 버튼)
    - 가로 드래그로 시간 이동, 스크롤로 확대·축소
    - **가중치 미적용 점수 vs 가중치 적용 점수**를 두 개의 라인으로 비교
    - 향후 안정성 방향(상승/유지/하락)을 화살표 아이콘으로 표현

- **③ 가중치 요인별 점수 패널**
  - 활동성 / 경제적 변동성 / 지속성 각각을 구성하는 세부 지표(예: 가격 변동성, NC 등)의 점수를 막대/레이더 차트 등으로 표현.

- **④ 블록체인 순위 보드**
  - 사용자가 설정한 가중치에 따른 전체 블록체인 랭킹
  - 각 체인의 총점, 세부 점수, 1·2위 간 점수 차이 등을 같이 보여줌
  - **순위 항목 클릭 시** 메인 그래프/디테일 패널의 대상 체인으로 선택되도록 연동

- **⑤ 요약 보드(Summary)**
  - 선택된 블록체인의
    - 안정성 총점
    - 안정성 추이 요약(최근 N일)
    - 현재 순위
  - 작은 카드/요약 그래프 형태로 한눈에 보이게 구성

### 6. UX 플로우(사용자 흐름)
1. 페이지 진입
2. 활동성/지속성/경제적 변동성에 대한 **가중치 설정**
3. 설정된 가중치로 계산된 **전체 블록체인 순위** 확인
4. 분석할 블록체인(랭킹 클릭 또는 드롭다운)과 기간 선택
5. 안정성 추이 그래프를 통해 시계열 데이터 분석
6. 요인별(활동성/경제적 변동성/지속성) 세부 점수 확인
7. 요약 보드에서 최종 결과 정리 확인

### 7. 교수님 피드백 요약(중요 포인트만)
- 질문이 더 명확해야 하고, 시각화가 그 질문에 직접 답하도록 구조를 다듬을 것.
- 지금 구성은 **나열형**이고, 특히 **추이 그래프와 점수 구조**의 깊이가 부족하다는 지적.
- **가중치는 전역 상태**이므로 페이지 **좌측 상단 또는 상단에 배치**하는 것이 맞다.
  - 제어부(가중치, 선택, 필터)와 결과부(그래프, 순위, 요약)를 분리할 것.
- **순위 보드**를 더 고도화:
  - 단순 순위만이 아니라 “몇 점인지, 무엇 때문에 차이가 나는지”까지 보여주기.
- Stability details(세부 안정성 지표)를 한눈에 볼 수 있게 별도 셀/뷰로 구성하면 좋다는 의견.
- 데이터 설계는 좋지만, **“데이터 처리는 잘했는데 결과 전달(UI/시각화)이 아쉽다”**는 총평.

### 8. 앞으로 정해야 할 것
- 각 세부 지표를 **0–100 점으로 정규화하는 방식(수식)** 설계
- 활동성/지속성/경제적 변동성 세 축 사이의 기본 가중치 초기값
- React + Tailwind 기준
  - 페이지 레이아웃(좌측 제어부 / 우측 결과부) 와이어프레임
  - `ChainScore`, `ActivityMetrics` 등 데이터 타입/인터페이스
  - 점수 계산 유틸 함수 구조

## 디자인 시스템 (Design System)

### 파일 구조
```
src/design/
├── tokens.js       # 디자인 토큰 정의
└── README.md       # 사용 가이드
```

### 핵심 컬러
- **배경**: `#121212` (메인), `#101010` (사이드바)
- **테두리**: `#292929`
- **텍스트**: `#ffffff` (주요), `#a1a9c0` (라벨), `#727c95` (placeholder)
- **브랜드**: `#455cdc` (로고)
- **차트**: 
  - Activity: `#f83464` (핑크/레드)
  - Economic Volatility: `#ff852f` (오렌지)
  - Persistence: `#43d2a7` (민트/그린)

### 타이포그래피
- **폰트**: Lato
- **크기**: 15px (기본), 20px (로고)
- **굵기**: 
  - Medium (500): 일반 텍스트
  - SemiBold (600): 체인 이름
  - Bold (700): 라벨, 섹션 타이틀
  - ExtraBold (800): 로고

### 레이아웃
- **Header 높이**: 60px
- **Sidebar 너비**: 450px
- **Border Radius**: 10px (기본)
- **Spacing**: 10px, 20px, 30px (주요 간격)
